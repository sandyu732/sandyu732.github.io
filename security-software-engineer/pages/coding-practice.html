Tier 1 — go deep

Security-themed coding practice

Google coding rounds are standard DS&A, not CTF-style — but for a security SWE role, practicing security-flavored problems builds fluency for scenario questions. ~20-30 min each, pseudocode and complexity stated out loud before coding.

Simple cipher
Implement a Caesar / XOR / simple substitution cipher, encode and decode.
Log parser
Extract IPs, timestamps, and domains from an arbitrary text blob using regex. Discuss the Big-O of your regex — catastrophic backtracking is a great thing to flag proactively.
Port scanner (either direction)
Write a basic port scanner, or detect port-scanning behavior from a log of connection attempts — same mental model.
Rate limiter / brute-force lockout
Implement sliding-window rate limiting — a natural bridge from coding into a security-design conversation.
JWT structural parser
Parse and validate a JWT's header.payload.signature, base64url decode, without a library. Good vehicle for discussing why you'd never do this for real without a vetted library, and what typically goes wrong — algorithm-confusion attacks like accepting alg: none.
Standard DS&A refresh
Hash tables, BSTs, two-pointer/sliding-window, BFS/DFS on graphs. Google's bar for coding rounds hasn't shifted, and depth in security topics doesn't substitute for fluency here.
Interview technique reminder
Write pseudocode first, state complexity, write your own test cases before running them mentally against your solution. See the interview technique cheat sheet.
