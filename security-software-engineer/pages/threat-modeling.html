Tier 2 — working knowledge

Threat modeling

STRIDE, DREAD, and the process behind them — plus a live tool to practice threat-modeling out loud against three scenarios.

Process

Architecture review → build a Data Flow Diagram → identify trust boundaries → enumerate threats per boundary/component using a framework → propose mitigations, prioritized by risk.

STRIDE

The framework most likely to come up. Know both the letter and the security property it violates.

LetterThreatViolates
SSpoofingAuthenticity
TTamperingIntegrity
RRepudiationAccountability
IInformation disclosureConfidentiality
DDenial of serviceAvailability
EElevation of privilegeAuthorization

Try it: pick a scenario, walk STRIDE out loud

Practice applying STRIDE live — talk through each letter against each major component before checking the example threat below.

DREAD

Damage, Reproducibility, Exploitability, Affected users, Discoverability. Know it, but also know it's considered somewhat obsolete — scores are ad hoc and subjective. Being able to say why it fell out of favor (and that structured risk discussion or CVSS largely replaced it) signals more maturity than reciting the acronym.

MITRE ATT&CK

The industry-standard matrix of adversary tactics/techniques. Useful less as trivia and more as shared vocabulary for describing an attack chain — see the attack structure page.
