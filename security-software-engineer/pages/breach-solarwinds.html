SolarWinds / Sunburst

A nation-state-attributed supply chain attack that turned a trusted, signed software update into the delivery mechanism for the backdoor itself.

Discovered by FireEye in December 2020 after the security firm noticed its own network had been breached — and traced the entry point back to its own SolarWinds Orion IT-monitoring software, attributed to Russia's SVR (tracked as APT29 / Nobelium).

Why signing didn't save anyone

Code signing is meant to prove a binary came from a trusted publisher and wasn't tampered with in transit. Sunburst broke that guarantee at the source: the malicious DLL was injected during SolarWinds' own legitimate build process, then signed with SolarWinds' own valid certificate along with the rest of the update. Every downstream trust check — "is this signed by SolarWinds?" — correctly said yes, because it genuinely was. The compromise was upstream of the control, not a forgery of it.

Patience as a tactic

The implant stayed dormant for roughly two weeks after installation, then communicated over DNS using a domain-generation pattern designed to blend into Orion's own normal telemetry traffic — deliberately avoiding a burst of suspicious activity right after install that monitoring might catch. Out of roughly 18,000 organizations that pulled the trojanized update, the attackers only progressed to actual hands-on-keyboard follow-up against a much smaller set of high-value targets, including multiple U.S. federal agencies.

Kerberos golden tickets, in practice

This is the real-world version of the concept
Once inside a target's Active Directory environment, attackers used forged Kerberos golden tickets — minted using a stolen krbtgt hash — to grant themselves domain-wide access without ever needing to touch the KDC again. See the mechanics on the auth & identity page.

Interview framing

If asked how to defend against this, the honest answer resists a single silver bullet: reproducible/verifiable builds, provenance attestation (SLSA-style), anomaly detection on the build environment itself (not just production), and network segmentation so that compromising one monitoring tool doesn't grant domain-wide reach.
