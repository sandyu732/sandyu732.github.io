Tier 3 — conversational fluency

Digital forensics

Evidence volatility, the three forensics domains, and why chain of custody matters — at a conversational level.

Evidence volatility

Highest
Network & memory — capture first
Medium
Disk — persists, but can be wiped
Lowest
Cold backups / archives

Network and memory state are the most volatile and time-sensitive to capture — a live memory dump is often gone the moment a box is rebooted, which is exactly why an IR runbook should specify memory acquisition before disk imaging in most cases.

The three domains

Memory forensics
Memory acquisition (footprint, smear, hiberfiles), virtual vs physical memory, kernel vs user space. Tools: Volatility, Google Rapid Response (GRR) / Rekall, WinDbg.
Disk forensics
Disk imaging, filesystems (NTFS / ext2-4 / APFS), logs (Windows event logs, Unix syslogs, application logs), data carving for recovery. Tools: plaso / log2timeline, FTK Imager, EnCase.
Mobile forensics
Jailbreaking implications, and how Android vs iPhone acquisition differs meaningfully from computer forensics.

Anti-forensics

Timestomping (altering file metadata timestamps to blend in or mislead investigators) is the classic example of malware actively working against the forensics process, not just against detection.

Chain of custody

Evidence integrity determines whether findings are usable at all — legally or organizationally. Handover notes at every stage of evidence handling are what make a chain of custody defensible.
